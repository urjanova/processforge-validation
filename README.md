# ProcessForge Validation

A tool for validating simulation results generated by ProcessForge. It verifies mass conservation, pump performance, and thermodynamic consistency, outputting a detailed Excel report.

## Features

- **Mass Validation**: Checks if chemical fractions sum to 1.0.
- **Equipment Checks**: Verifies pump pressure gains and thermal consistency.
- **Flexible Input**: Supports local Zarr folders, S3 URLs (`s3://`), HTTP(S) zipped archives (`http://`, `https://`), and env-var-driven configuration via `LOCAL_ZARR_DIR` or `S3_BUCKET_NAME`.
- **Comprehensive Reporting**: Generates a multi-sheet Excel file with an executive summary and detailed data.

## Installation

This package requires Python 3.12+ and the dependencies listed in `pyproject.toml`.

1. Clone the repository:
   ```bash
   git clone https://github.com/urjanova/processforge-validation.git
   cd processforge-validation
   ```

2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   # OR use the project file directly if using a tool like poetry/pdm, or install manually:
   pip install numpy pandas zarr boto3 requests openpyxl loguru coolprop matplotlib
   ```

## Configuration

Copy `.env.example` to `.env` and fill in the values for your environment.

| Variable | Description |
|---|---|
| `LOCAL_ZARR_DIR` | Path to a local folder of Zarr files. Takes priority over S3 when set. |
| `S3_ACCESS_KEY` | S3-compatible access key. |
| `S3_SECRET_KEY` | S3-compatible secret key. |
| `S3_REGION_NAME` | Region (e.g. `ams3` for DigitalOcean Spaces). |
| `S3_ENDPOINT_URL` | Custom S3 endpoint URL (for non-AWS providers). |
| `S3_BUCKET_NAME` | Bucket name used as the default source when no argument is passed. |

> **Warning**: If any `S3_*` variables are missing when an S3 source is used, a warning is logged and the connection may fail. Use `LOCAL_ZARR_DIR` as a credential-free alternative.

## Usage

```bash
python validate.py [SOURCE] [-o OUTPUT]
```

### Arguments

- `SOURCE` *(optional)*: The input Zarr store. Can be:
  - A local directory path (e.g., `./my_simulation.zarr`).
  - An S3 URL (e.g., `s3://my-bucket/simulations/run1.zarr`).
  - An HTTP/HTTPS URL pointing to a `.zip` archive of a Zarr store.
  - If omitted, `LOCAL_ZARR_DIR` is used first, then an S3 URL built from `S3_BUCKET_NAME`.
- `-o`, `--output`: Path for the output Excel report. Defaults to `validation.xlsx`.

### Examples

**Local directory (explicit path)**
```bash
python validate.py ./dataset.zarr
```

**Local directory (via env var — no argument needed)**
```bash
LOCAL_ZARR_DIR=./dataset.zarr python validate.py
```

**S3 URL**
```bash
python validate.py s3://processforge-data/run_01.zarr -o report_01.xlsx
```

**S3 via env var — no argument needed**
```bash
# S3_* vars set in .env
python validate.py
```

**HTTP zip archive**
```bash
python validate.py https://example.com/data/sim_result.zip
```

## Output

The generated Excel file contains:
1.  **1_EXECUTIVE_SUMMARY**: High-level pass/fail status for physical laws (Mass Conservation, Pump Work, etc.).
2.  **2_PUMP_PERFORMANCE**: Detailed breakdown of pressure and temperature changes across pumps (only if pumps are detected).
3.  **3_RAW_DATA_CHECKED**: The full processed dataset with added validation columns (`Total_Fraction`, `Composition_Alert`).
